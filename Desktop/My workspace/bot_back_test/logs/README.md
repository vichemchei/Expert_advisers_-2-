# Trading Bot Logs

This directory contains all log files generated by the trading bot system. The logs are organized by bot type and date to help with debugging, performance analysis, and trade monitoring.

## Log File Types

### 1. Bot Diagnostic Logs
**Pattern:** `bot_diagnostic_YYYYMMDD.log`

These logs contain detailed diagnostic information about the bot's operations, including:
- System health checks
- Configuration validation
- Error diagnostics
- Performance metrics
- Memory and resource usage

- `bot_diagnostic_20251016.log`
- `bot_diagnostic_20251017.log`

## CSV Data Files

### signals.csv
Contains all trading signals generated by the bot:
- Signal timestamp
- Signal type (buy/sell)
- Price levels
- Indicator values
- Confidence scores

### trades.csv
Records all executed trades:
- Trade entry and exit times
- Entry and exit prices
- Position size
- Profit/loss
- Trade duration

### daily_performance.csv
Daily performance metrics:
- Daily P&L
- Win rate
- Number of trades
- Maximum drawdown
- Sharpe ratio

### errors.csv
Error tracking and logging:
- Error timestamps
- Error types
- Error messages
- Stack traces
- Resolution status

## Log Rotation

Logs are automatically created daily with the naming pattern `[bot_type]_YYYYMMDD.log`. This ensures:
- Easy identification of logs by date
- Manageable file sizes
- Historical data preservation
- Simplified debugging for specific dates

## Usage Tips

### Viewing Recent Activity
```powershell
# View the last 50 lines of today's FVG BOS bot log
Get-Content fvg_bos_bot_$(Get-Date -Format "yyyyMMdd").log -Tail 50
```

### Searching for Errors
```powershell
# Search for errors in all logs
Select-String -Path "*.log" -Pattern "ERROR|EXCEPTION|FAILED"
```

### Analyzing Trade Performance
```powershell
# View trades CSV
Import-Csv trades.csv | Format-Table
```

### Monitoring Real-time Logs
```powershell
# Monitor today's bot log in real-time
Get-Content bot_$(Get-Date -Format "yyyyMMdd").log -Wait -Tail 20
```

## Log Retention

- **Active logs:** Last 30 days kept in this directory
- **Archived logs:** Older logs should be compressed and moved to archive folder
- **Critical logs:** Logs with significant events should be backed up separately

## Troubleshooting

### Common Issues

1. **Large log files:** If log files exceed 100MB, check for excessive logging or errors causing repeated messages
2. **Missing logs:** Verify bot is running and has write permissions to this directory
3. **Corrupted logs:** Check disk space and file system integrity

### Log Levels

The bot uses the following log levels:
- **DEBUG:** Detailed information for diagnosing problems
- **INFO:** General informational messages
- **WARNING:** Warning messages for potentially harmful situations
- **ERROR:** Error events that might still allow the application to continue
- **CRITICAL:** Severe errors causing the application to abort
